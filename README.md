## What I Learned in Lesson 0, Day 0:
* **python/py:** Computer only understands binary i.e. 0s and 1s, so the "py" called in terminal will translate to computer in the computer language 0s & 1s then translate the output to human language. 

* **String:** a string is a sequence of characters used to represent text. Anything inside single quotes ('...'), double quotes ("..."), or triple quotes ("""...""") is considered a string. Learned how running a program in the terminal using `py hello.py` provides the output called "side effect".

* **Side Effects:** In programming, a "side effect" is an observable change that happens when a line of code runs. For example, using `print()` causes the side effect of displaying text in the terminal window. It is the result of your code that shows in the terminal

* **Function:** functions are reusable code. Functions are actions or verbs that the computer or computer language already know how to use which aids we humans to use the function codes to execute from the computer. i.e. *print*

* **Arguments:** are Information that passes through functions to display as side effects. it somehow influences functions i.e. *print as in **function***, *`("hello,world")` as in argument* `print("hello, world.")`
*hint:* the parentheses and the double quote made it possible for argument to display exactly what's in it.

* **Variables:** variables have the ability to store information in the computer memories. Practiced using the `input()` function to capture variables (like names, ages, and weights) and `print()` to output them.
*hint:* To use "print" to output the information stored, the double quotes will be excluded.

* **String Formatting:** Tested spacing using both the `+` concatenation operator and commas `,` for automatic spacing, another argument wants to be added.
